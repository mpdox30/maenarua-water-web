# สรุปไอเดียเว็บไซต์: ศูนย์ข้อมูลสนับสนุนการบริหารจัดการน้ำ ตำบลแม่นาเรือ

**พื้นที่:** ตำบลแม่นาเรือ อำเภอเมือง จังหวัดพะเยา
**วันที่สรุป:** 3 กรกฎาคม 2569

---

## 1. วัตถุประสงค์เว็บไซต์

รวมงานวิจัยด้านน้ำที่ทำอยู่ทั้งหมด ได้แก่ การพยากรณ์ความต้องการใช้น้ำ, การพยากรณ์น้ำไหลลงอ่าง, และ water monitoring จากสถานีโทรมาตร เข้าเป็นศูนย์ข้อมูลเดียว เพื่อสนับสนุนการตัดสินใจของเกษตรกรและหน่วยงานท้องถิ่น

---

## 2. โครงสร้างหลักของเว็บไซต์

1. **หน้าแรก** — ภาพรวมพื้นที่, แผนที่ Zone A/B, KPI card สรุป
2. **การพยากรณ์ความต้องการใช้น้ำ** — แยกตาม Zone A (NIR_A rainfed) / Zone B (GID_B irrigated)
3. **การพยากรณ์น้ำไหลลงอ่าง (Inflow Forecasting)**
4. **Water Monitoring** — ข้อมูลจากสถานีโทรมาตร (อากาศ + ระดับน้ำในอ่าง)
5. **แผนที่เชิงพื้นที่ (GIS)** — โซนเกษตร, อ่างเก็บน้ำ, คลองส่งน้ำ (แคววังทอง)
6. **งานวิจัย/เอกสารเผยแพร่**

## 3. การจัดหมวดข้อมูล

| หมวด | เนื้อหา | เหตุผล |
|---|---|---|
| **สาธารณะ** | กราฟสรุปพยากรณ์รายสัปดาห์, แผนที่, % ความจุอ่าง, ข่าวสาร/สรุปงานวิจัย | สร้างความเข้าใจ ให้วางแผนเบื้องต้นได้ |

## 4. สถานะความพร้อมของโมเดลพยากรณ์

- **ยังไม่พร้อมเป็น live forecast เต็มรูปแบบ**
- ปัจจุบันเป็นการ validate ย้อนหลังเท่านั้น (train 2020–2022, calibrate 2023, test 2024)
- Results ยัง pending experimental completion
- ต้องเพิ่ม automated pipeline ก่อนใช้งานจริงแบบ live

---

## 5. แนวทาง Live Forecast (เก็บไว้สำหรับพัฒนาต่อในอนาคต)

- รันโมเดลพยากรณ์ทุกสัปดาห์ตามรอบที่ตั้งใจแสดงผล ไม่ fix รอที่ 3 สัปดาห์ตายตัว
- แยก cadence การอัปเดตตามชนิดข้อมูล:
  - โทรมาตร: รายชั่วโมง/รายวัน
  - Sentinel-1 SAR: ทุก ~7–10 วัน (ตาม revisit cycle)
  - MODIS ET: ทุก ~2 สัปดาห์ พร้อม gap-filling สำหรับช่วงที่ latency ค้าง
- ใช้หลัก "best-available input" ในแต่ละรอบพยากรณ์
- แสดง timestamp ของข้อมูลล่าสุดชัดเจนบนเว็บ เพื่อไม่ให้เข้าใจผิดว่าเป็นระบบ real-time เป๊ะ

## 6. Tools / Tech Stack

> หมายเหตุ: ตัดตัวเลือก HII infrastructure ออก เนื่องจากไม่มีสิทธิ์เข้าใช้งาน

| เลเยอร์ | เครื่องมือ | ค่าใช้จ่าย |
|---|---|---|
| Data ingestion | GEE Python API, Cloud Scheduler | ฟรี (research tier) / ถูกมาก |
| Storage | PostgreSQL + TimescaleDB, PostGIS | ต้องเลือก hosting เพิ่มเติม |
| Modeling | scikit-learn, CatBoost, LightGBM, MAPIE | ฟรี 100% |
| Model serving | FastAPI + Docker + Cloud Run | Free tier (2M requests/เดือน) |
| Frontend | D3.js, Leaflet, GitHub Pages | ฟรี 100% |
| Monitoring | Grafana Cloud (free tier), MLflow | ฟรี / free tier |
| Authentication | Firebase Auth | ฟรีสำหรับผู้ใช้จำนวนน้อย–กลาง |

### ทางเลือก Hosting ที่ต้องพิจารณาต่อ
- Cloud Run + Cloud SQL (Google) — free tier + pay-as-you-go เมื่อเกิน
- Railway / Render — free tier เหมาะกับ prototype/demo
- เซิร์ฟเวอร์ของมหาวิทยาลัยพะเยา (ถ้ามีสิทธิ์เข้าถึง) — ควรเช็คเป็นตัวเลือกถัดไป
